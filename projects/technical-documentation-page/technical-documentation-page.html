Introduction

React is a declarative, efficient, and flexible JavaScript library for building user interfaces. It lets you compose complex UIs from small and isolated pieces of code called "components".

React has a few different kinds of components, but we'll start with React.Component subclasses.

import React from 'react'; import ReactDOM from 'react-dom/client';
Core Concepts

React operates on a few core principles that make it powerful and intuitive to use once you understand the paradigms.

  • Components
  • JSX (JavaScript XML)
  • Props (Properties)

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation.

const element = <h1>Hello, world!</h1>;
Hooks

Hooks are a new addition in React 16.8. They let you use state and other React features without writing a class.

  • useState
  • useEffect

You can also build your own Hooks to share reusable stateful logic between different components.

import React, { useState } from 'react'; function Example() { const [count, setCount] = useState(0); return ( <div> <p>You clicked {count} times</p> <button onClick={() => setCount(count + 1)}> Click me </button> </div> ); }
Components

Components are the fundamental building blocks of any React app. A typical React app therefore has many of these.

Conceptually, components are like JavaScript functions. They accept arbitrary inputs (called "props") and return React elements describing what should appear on the screen.

function Welcome(props) { return <h1>Hello, {props.name}</h1>; }
State Management

State management is an important part of any robust React application. As your application grows, managing the state between components becomes crucial.

For simple apps, React's built-in state might suffice. However, for larger applications, you might consider libraries like Redux or the Context API.

const ThemeContext = React.createContext('light'); const value = useContext(ThemeContext);
Routing

Routing in modern React applications is typically handled by third-party libraries like React Router.

It allows you to map different URLs to different components, enabling navigation without fully reloading the page.

import { BrowserRouter, Route, Switch } from 'react-router-dom'; <BrowserRouter> <Switch> <Route path="/about" component={About} /> <Route path="/" component={Home} /> </Switch> </BrowserRouter>
